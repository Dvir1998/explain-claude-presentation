## עבודה עם שיחות

### פתיחת שיחה חדשה
```bash
# בתיקייה שלך
claude

# עם שאלה ראשונית
claude "הסבר לי את הקוד הזה"
```

### המשך שיחה אחרונה
```bash
claude --continue
# או בקיצור:
claude -c
```

### בחירת שיחה מהיסטוריה
```bash
claude --resume
# או בקיצור:
claude -r

# לשיחה ספציפית
claude -r "שם-השיחה"
```

### עבודה עם כמה שיחות במקביל

1. **פתח טאב חדש:** `Cmd+T` (Mac) או `Ctrl+Shift+T`
2. **עבור לתיקייה אחרת:** `cd /other/project`
3. **פתח Claude:** `claude`

עכשיו יש לך שתי שיחות נפרדות!

### ניווט ב-Session Picker (תוך `/resume`)

| מקש | פעולה |
|-----|-------|
| **↑ / ↓** | עבור בין שיחות |
| **→ / ←** | פתח/סגור קבוצות |
| **Enter** | בחר שיחה |
| **P** | תצוגה מקדימה |
| **R** | שנה שם |
| **/** | חפש |
| **Esc** | יציאה |

## אפשרויות שורת פקודה

### הכי חשובות

| פקודה | מה עושה |
|-------|---------|
| `claude` | שיחה חדשה |
| `claude -c` | המשך שיחה אחרונה |
| `claude -r` | בחר מהיסטוריה |
| `claude -p "שאלה"` | שאל בלי אינטראקציה |
| `claude --model opus` | השתמש במודל Opus |
| `claude --model haiku` | השתמש במודל Haiku |

### מתקדמות

| פקודה | מה עושה |
|-------|---------|
| `claude --permission-mode plan` | מצב תכנון בלבד |
| `claude --add-dir ../other` | הוסף תיקייה נוספת |
| `claude --max-turns 5` | הגבל מספר תורים |
| `claude --max-budget-usd 5` | הגבל תקציב |
| `claude --verbose` | הצג פלט מפורט |
| `claude --version` | הצג גרסה |

### דוגמאות שימוש

```bash
# שאל שאלה מהירה
claude -p "מה הגרסה של Node.js?"

# המשך עם מודל חזק יותר
claude --continue --model opus

# עבוד במצב תכנון
claude --permission-mode plan

# הרץ עם תקציב מוגבל
claude -p "analyze code" --max-budget-usd 2.00
```

## הגדרות

### קבצי הגדרות

| קובץ | מיקום | השפעה |
|------|-------|-------|
| **User** | `~/.claude/settings.json` | כל הפרויקטים שלך |
| **Project** | `.claude/settings.json` | פרויקט ספציפי |
| **Local** | `.claude/settings.local.json` | רק אתה בפרויקט |

### CLAUDE.md - זיכרון הפרויקט

קובץ `CLAUDE.md` נטען בכל שיחה. שמור בו הוראות חשובות:

```markdown
# Build Commands
- npm run build - בנה פרויקט
- npm run test - הרץ בדיקות

# Code Style
- השתמש ב-TypeScript
- כתוב קוד נקי

# חשוב לזכור
- הקפד על RTL בעברית
- השתמש ב-Gemini 3 Pro
```

ליצור אוטומטית:
```bash
/init
```
